# Ethereum Development & Orchestration

Specialized expertise in smart contract development, orchestration, and testing. Based on the `ethskills.com` standard.

## Procedural Guidance

### Smart Contract Tooling (`eth-tools`)
- **Foundry Master**: Mastery of Foundry (`forge`, `cast`, `anvil`) for fast compilation, testing, and deployment.
- **Scaffold-ETH 2**: Rapid prototyping using the Scaffold-ETH 2 framework (Viem, Wagmi, Next.js).
- **RPC Configuration**: Manage and optimize RPC endpoints for latency and reliability.
- **MCP (Model Context Protocol)**: Integrate AI agents using MCP standards.

### Standards & Legos (`eth-standards`)
- **Token Standards**: Expert implementation of ERC-20, ERC-721, and ERC-1155.
- **Agent Identity**: Implement EIP-8004 (Agent Identity) and EIP-7702 (Smart EOAs).
- **DeFi Composability**: Build with "Money Legos" like Uniswap, Aave, and Compound.

### Testing & Validation (`eth-testing`)
- **Unit Testing**: Write granular tests for contract logic using `forge test`.
- **Fuzz Testing**: Use property-based testing to find edge cases.
- **Fork Testing**: Test contract interactions against live Mainnet state using Foundry fork mode.
- **Invariant Testing**: Define and test global system invariants.

### Orchestration (`eth-orchestration`)
- **3-Phase Build System**: Implement standard Research -> Strategy -> Execution pipelines.
- **dApp Orchestration**: Define end-to-end deployment scripts and automated pipelines.
